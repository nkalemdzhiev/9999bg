# File Ownership Map

This document maps every file in the project to an engineer. **Do not edit another engineer's files without coordinating first.** This minimizes merge conflicts during the hackathon.

## Merge Rules

1. **Engineer 1** is the only one who edits `router.ex` during initial setup.
2. **Engineer 2** can add new routes to `router.ex` but must not rename existing ones.
3. **Engineer 3** never touches `router.ex`.
4. If two engineers need the same file, the one who owns the **module** in that file edits it; the other adds a helper in their own module.
5. Commit after every file ownership boundary is crossed.
